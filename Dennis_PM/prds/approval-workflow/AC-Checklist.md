# 验收标准清单 (AC Checklist)

> **关联PRD**: PRD-APPROVAL-001 V1.0
> **日期**: 2026-03-19
> **用途**: 供QA/开发团队验收测试使用

## APPROVAL-001 审批流引擎

### 正向流程
- [ ] AC-001 申请发起后，approval_request.status = 20（审批中），template_snapshot 已写入完整模板JSON
- [ ] AC-002 第一个节点的审批人收到站内消息和邮件通知
- [ ] AC-003 会签节点：所有审批人通过后，approval_request 推进到下一节点
- [ ] AC-004 或签节点：任一审批人通过后，approval_request 推进到下一节点
- [ ] AC-005 最后节点全部通过后，approval_request.status = 30（已通过），passed_at 已记录
- [ ] AC-006 任一审批人拒绝后，approval_request.status = 40（已拒绝），拒绝原因已记录，申请人收到通知

### 撤回流程
- [ ] AC-007 审批中且当前节点无人操作时，申请人可撤回；撤回后 status = 50（已撤回）
- [ ] AC-008 当前节点已有人操作时，撤回按钮不可用，提示「审批进行中不可撤回」
- [ ] AC-009 撤回后所有 PENDING 的 approval_instance.status = 50（已跳过），待审批人收到撤回通知

### 并发安全
- [ ] AC-010 两个审批人同时对会签节点操作，最终节点状态正确，不出现重复推进
- [ ] AC-011 审批人通过与申请人撤回同时发生，乐观锁校验失败方返回409错误提示
- [ ] AC-012 同一业务单据重复提交，第二次提交返回错误「该单据已存在进行中的申请」

### 委托代理
- [ ] AC-013 审批人设置委托后，新审批任务分配给代理人而非原审批人
- [ ] AC-014 审批记录中 is_delegated = true，delegator_id = 原审批人ID
- [ ] AC-015 委托到期后，新审批任务恢复分配给原审批人

## APPROVAL-002 审批流配置后台

- [ ] AC-016 Super Admin 可创建审批流模板，配置多个节点、角色、会签/或签、金额条件
- [ ] AC-017 普通员工访问审批流配置接口返回403
- [ ] AC-018 停用模板后，发起对应类型申请时提示「审批流未配置，请联系管理员」
- [ ] AC-019 编辑模板后，存量进行中申请继续按原快照执行，不受影响
- [ ] AC-020 同一 biz_type 尝试启用第二个模板时，系统提示「该类型已有启用模板」

## APPROVAL-003 约车付款申请

- [ ] AC-021 约车账单状态=已生成时，详情页显示「发起付款申请」按钮
- [ ] AC-022 申请金额输入超过 $1,000 USD，前端实时提示错误，提交按钮禁用
- [ ] AC-023 后端接口校验金额超过 $1,000 USD，返回 400 错误
- [ ] AC-024 审批通过后，约车账单状态自动更新为「付款审批通过」，无需人工操作
- [ ] AC-025 同一约车账单发起第二条付款申请时，系统提示「该账单已存在进行中的付款申请」
- [ ] AC-026 财务标记「已付款」后，约车账单状态更新为「已付款」，操作人和时间已记录

## APPROVAL-004 客户开票申请

- [ ] AC-027 OMS充值记录列表，已付款记录显示「申请开票」按钮，未付款记录不显示
- [ ] AC-028 首次发起开票申请，客户必须录入开票信息（抬头/税号/地址/电话/开户行/账号）
- [ ] AC-029 二次发起时，自动预填上次的开票信息，客户可修改
- [ ] AC-030 后端校验：开票申请 amount 必须等于对应充值记录的实付金额，不一致返回400
- [ ] AC-031 同一充值记录重复发起开票申请，系统提示「该记录已存在开票申请」
- [ ] AC-032 财务上传发票文件后，客户收到站内消息和邮件通知
- [ ] AC-033 客户点击下载发票，后端生成有效期1小时的S3签名URL，超时后链接失效
- [ ] AC-034 客户A无法访问客户B的开票申请和发票，返回403
- [ ] AC-035 「我的发票」页面展示所有历史申请状态和可下载发票列表

## APPROVAL-007 审批委托

- [ ] AC-043 审批人可设置委托：被委托人、开始时间、结束时间、委托业务类型
- [ ] AC-044 设置委托人和被委托人为同一人时，提示错误
- [ ] AC-045 A委托B后，尝试B委托A，系统提示「存在反向委托，无法创建」

- [ ] AC-047 委托到期后（超过end_at），is_active自动变为false，新任务不再分配给代理人

## APPROVAL-008 审批消息通知
