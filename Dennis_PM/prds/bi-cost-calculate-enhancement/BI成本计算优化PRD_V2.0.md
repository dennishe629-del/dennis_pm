## 背景

### 业务背景

仓盛海外仓主要业务模式：
- **一件代发**：主要业务模式，小件客户居多（占80%+）
- **转运**：货物在不同仓库间的转运
- **入库**：客户货物入库操作
- **退件**：退货处理
- **存储**：货物仓储
- **增值服务**：拍照、贴标、二次包装等

**仓库分布**：美国自有仓库4个（美东、美西等）
**客户规模**：100-200个客户

### 核心问题：人工成本误差高达20%

**问题表现**：
- 操作费、入库费、退件费按仓库汇总后，与财务录入的实际人工成本差异约20%
- 成本报价需要频繁调整，但调整后误差依然存在
- 客户维度和订单维度的成本数据对不上账

**现有计算流程**：
```
步骤1：按销售额占比分摊人工成本
  仓库_入库费人工成本 = 仓库_实际人工成本 × (入库费收入 / 总收入)
  仓库_操作费人工成本 = 仓库_实际人工成本 × (操作费收入 / 总收入)
  仓库_退件费人工成本 = 仓库_实际人工成本 × (退件费收入 / 总收入)

步骤2：根据成本报价计算预估人工成本
  仓库_预估人工成本 = Σ(所有单据的成本报价)

步骤3：计算调整系数
  调整系数 = 仓库_实际人工成本 / 仓库_预估人工成本

步骤4：更新成本报价
  新成本报价 = 原成本报价 × 调整系数

步骤5：重新计算上月成本
  使用新成本报价重新计算所有单据成本
```

### 根本原因分析

**1. 循环依赖导致误差累积**
- 成本报价 → 预估成本 → 调整系数 → 新成本报价 → 新预估成本
- 每次调整都引入新误差，多次调整后误差累积到20%

**2. 分摊逻辑不准确**
- 按销售额占比分摊，但销售额占比≠人工成本消耗占比
- 举例：入库费收入高但操作简单（人工成本低），操作费收入低但操作复杂（人工成本高）

**3. 未考虑单据复杂度**
- 同类单据使用统一成本报价
- 小件出库单（1个SKU）vs 大件出库单（50个SKU）成本相同，不合理

**4. 多次计算放大误差**
- 4个步骤，每步都引入误差，最终误差累积到20%

## 优化方案

### 核心创新：基于单据复杂度的成本分摊

**新计算流程**：
```
步骤1：获取月度人工成本
  仓库_人工成本_total = 财务录入的月度人工成本

步骤2：计算每个单据的复杂度得分
  单据_复杂度得分 = f(SKU数量, 重量, 体积, 增值服务等)
  得分范围：0-1

步骤3：确定单据复杂度等级和成本系数
  L1 (0-0.2): 成本系数 0.5 (极简单)
  L2 (0.2-0.4): 成本系数 0.8 (简单)
  L3 (0.4-0.6): 成本系数 1.0 (中等)
  L4 (0.6-0.8): 成本系数 1.3 (复杂)
  L5 (0.8-1.0): 成本系数 1.8 (极复杂)

步骤4：计算单据基准成本
  单据_基准成本 = 基础成本报价 × 单据_成本系数

步骤5：计算单据分摊系数
  单据_分摊系数 = 单据_基准成本 / Σ(所有单据_基准成本)

步骤6：分摊人工成本到单据
  单据_人工成本 = 仓库_人工成本_total × 单据_分摊系数

步骤7：验证分摊结果
  验证：Σ(所有单据_人工成本) = 仓库_人工成本_total
```

### 优势对比

| 对比项 | 现有逻辑 | 优化逻辑 | 改进效果 |
|-------|---------|---------|---------|
| 分摊依据 | 销售额占比 | 单据复杂度 | 更准确反映人工成本消耗 |
| 计算次数 | 2次（预估→调整→重算） | 1次 | 避免误差累积 |
| 是否考虑复杂度 | 否 | 是 | 提升精度 |
| 是否循环依赖 | 是 | 否 | 消除循环 |
| 误差范围 | 20% | 目标2% | 大幅提升 |

### 优化目标

1. **准确性**：成本误差从20%降低到2%以内（核心目标）
2. **实时性**：订单完成后5分钟内完成成本计算
3. **稳定性**：消除循环依赖，成本报价保持稳定
4. **可追溯**：每笔成本可追溯到复杂度计算过程
5. **可扩展**：支持新增复杂度因素和调整权重

### M02: 单据复杂度计算

**功能概述**：计算每个单据的复杂度得分，用于准确分摊人工成本

#### 复杂度因素定义

**入库单复杂度因素**：
- SKU数量：SKU越多，入库越复杂
- 重量：重量越大，入库越复杂
- 体积：体积越大，入库越复杂
- 包装类型：散装/托盘，复杂度不同

**出库单复杂度因素**：
- SKU数量：SKU越多，拣货越复杂
- 订单行数：订单行数越多，打包越复杂
- 重量、体积
- 增值服务：有增值服务的单据更复杂

**转运单复杂度因素**：
- SKU数量、重量、体积、转运距离

**增值服务单复杂度因素**：
- 服务类型：拍照/贴标/二次包装等
- SKU数量、重量

**退货单复杂度因素**：
- SKU数量、重量、体积
- 退货原因：质量问题/客户拒收等

#### 复杂度计算公式

**入库单复杂度得分**：
```
入库单_复杂度得分 = 
    (SKU数量 / max_SKU数量) × 30% +
    (重量 / max_重量) × 25% +
    (体积 / max_体积) × 25% +
    包装类型_系数 × 20%
```

**出库单复杂度得分**：
```
出库单_复杂度得分 = 
    (SKU数量 / max_SKU数量) × 25% +
    (订单行数 / max_订单行数) × 15% +
    (重量 / max_重量) × 20% +
    (体积 / max_体积) × 20% +
    增值服务_系数 × 20%
```

#### 复杂度等级划分

| 等级 | 得分范围 | 成本系数 | 说明 |
|-----|---------|---------|------|
| L1 | 0-0.2 | 0.5 | 极简单 |
| L2 | 0.2-0.4 | 0.8 | 简单 |
| L3 | 0.4-0.6 | 1.0 | 中等（基准） |
| L4 | 0.6-0.8 | 1.3 | 复杂 |
| L5 | 0.8-1.0 | 1.8 | 极复杂 |

### M03: 人工成本分摊引擎（核心）

**功能概述**：根据单据复杂度，将人工成本准确分摊到每个单据

**分摊逻辑**：

```python
# 步骤1：获取月度人工成本
warehouse_labor_cost = get_monthly_labor_cost(warehouse_id, month)

# 步骤2：获取上月所有单据
orders = get_orders(warehouse_id, month)

# 步骤3：计算每个单据的复杂度得分
for order in orders:
    order.complexity_score = calculate_complexity(order)
    
# 步骤4：确定复杂度等级和成本系数
for order in orders:
    order.complexity_level = get_level(order.complexity_score)
    order.cost_coefficient = get_coefficient(order.complexity_level)

# 步骤5：计算单据基准成本
for order in orders:
    order.base_cost = base_price × order.cost_coefficient

# 步骤6：计算单据分摊系数
total_base_cost = sum(order.base_cost for order in orders)
for order in orders:
    order.allocation_ratio = order.base_cost / total_base_cost

# 步骤7：分摊人工成本到单据
for order in orders:
    order.labor_cost = warehouse_labor_cost × order.allocation_ratio

# 步骤8：验证分摊结果
assert sum(order.labor_cost for order in orders) == warehouse_labor_cost
```

**业务规则**：
- 分摊系数总和必须为100%
- 单据人工成本不能为负数
- 分摊结果必须验证总和一致性
- 支持按仓库分别分摊

### M04: 成本差异分析

**功能概述**：分析成本差异，提供差异告警和优化建议

**差异计算**：
```
仓库_入库费人工成本 = Σ(上月所有入库单_人工成本)
仓库_操作费人工成本 = Σ(上月所有出库单/转运单/增值服务单_人工成本)
仓库_退件费人工成本 = Σ(上月所有退货单_人工成本)
仓库_总人工成本 = 三者之和

仓库_差异 = 仓库_总人工成本 - 仓库_实际人工成本
仓库_差异比例 = |仓库_差异| / 仓库_实际人工成本 × 100%
```

**告警规则**：
- 差异比例 < 2%：正常
- 差异比例 2%-5%：黄色告警
- 差异比例 5%-10%：橙色告警
- 差异比例 > 10%：红色告警

1. 成本差异比例 < 2%（核心指标）

3. 仓库维度成本汇总与实际人工成本差异 < 2%

**测试方法**：
- 使用真实历史数据进行回归测试
- 对比新旧系统的计算结果
- 财务团队验证成本准确性

## 风险评估

### 附录 A: 复杂度计算示例

**示例1：简单出库单（L1）**
- SKU数量：1个
- 订单行数：1行
- 重量：0.5kg
- 体积：0.01立方米
- 增值服务：无

复杂度得分 = (1/50)×25% + (1/20)×15% + (0.5/100)×20% + (0.01/5)×20% + 0×20% = 0.015
复杂度等级：L1
成本系数：0.5

**示例2：复杂出库单（L4）**
- SKU数量：30个
- 订单行数：15行
- 重量：50kg
- 体积：2立方米
- 增值服务：有（拍照+贴标）

复杂度得分 = (30/50)×25% + (15/20)×15% + (50/100)×20% + (2/5)×20% + 1×20% = 0.7
复杂度等级：L4
成本系数：1.3
