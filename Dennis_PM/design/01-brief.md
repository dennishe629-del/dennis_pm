# 设计目标与原则

> 适用范围：OMS / WMS / POD Design Studio / 财务 / BI 等 B 端后台模块。

## 设计目标

- 支持高频录入、检索、审核、对账、追踪等后台作业
- 在高信息密度前提下保持清晰层次
- 通过统一组件规范降低研发与维护成本

## 设计原则

| 原则 | 说明 |
|------|------|
| 任务优先 | 关键操作路径最短 |
| 状态可见 | 任何流程节点可追踪、可定位 |
| 一致性 | 跨模块视觉与交互一致 |
| 可扩展 | 支持模块扩展与多角色差异化 |

## 设计对象

- 运营后台（ShipSage Admin）
- 商家后台（Merchant Admin）
- 仓库作业界面（WMS）
- 财务管理后台
- BI 看板

## 设计输出物

- 信息架构
- 页面原型
- 交互说明
- 视觉规范

## 规范版本

本规范基于**生产系统截图分析**，优先贴近现有系统组件规范。

> 详细规范见 `design/frontend-standards.md`
