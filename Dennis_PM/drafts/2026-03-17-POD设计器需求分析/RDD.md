## 一、核心洞察 (Insight)

### 真实痛点

当前POD行业存在三层断裂：

1. **商家侧断裂**：Printful/Printify只支持商家预设固定款，消费者只能「选款」不能「定制」，错失个性化溢价空间（定制款比固定款溢价30-80%）
2. **消费者侧断裂**：有个性化购买意愿，但缺乏简单易用的店内定制入口，要么跳转第三方平台（摩擦大、流失高），要么只能接受标准款
3. **仓盛侧断裂**：已有美国8仓+成熟仓配能力，但缺乏前端流量入口，POD业务无法启动，现有能力无法变现

### JTBD（待办任务）

| 角色 | 想完成的任务 | 以便达成的目标 |
|------|------------|---------------|
| **商家（Shopify卖家）** | 在店铺里提供定制商品，不需要自己管理库存和生产 | 提高客单价和差异化竞争力 |
| **消费者** | 在购买时把自己的照片/文字印在商品上 | 获得独一无二、有个人意义的商品 |
| **仓盛运营** | 接收包含印刷文件的标准化订单，自动触发生产流程 | 实现POD业务规模化、低人工干预履约 |

### 业务价值评估

| 维度 | 评估 | 说明 |
|------|------|------|
| 战略价值 | **High** | POD是仓盛从「纯仓配」向「仓配+前端服务」转型的关键入口 |
| 收入潜力 | **High** | 每件定制商品毛利$13-19，比纯仓配出库费高5-10倍 |
| 竞争差异化 | **High** | 美国本土生产+2-4天交期，行业无对标竞品 |
| 开发复杂度 | **Medium** | 设计编辑器前端复杂，后端可复用ShipSage OMS |
| 市场时机 | **High** | TikTok Shop+AI内容驱动个性化需求爆发期 |

## 二、用户故事 (User Story)

### 主线故事1：商家配置定制商品

> **As a** Shopify独立站卖家（月销$5k-50k）
> **I want to** 在Shopify后台安装App，选择品类、上传设计模板、设置可定制区域和定价，一键同步到店铺
> **So that** 店铺能出售个性化定制商品，消费者完成定制后自动下单，我不需要手动处理印刷文件和发货

**典型操作路径**：
1. 安装App → 选择「T恤」品类
2. 上传3个设计模板作为起点
3. 设置「正面印刷区域」允许消费者修改，背面固定品牌Logo
4. 定价：基础款$29，定制加收$5
5. 一键同步到Shopify商品列表

### 主线故事2：消费者在店铺内完成定制

> **As a** 在Shopify独立站购物的消费者
> **I want to** 点击商品页的「自定义设计」按钮，在弹出的编辑器里上传照片、输入文字、实时预览效果，直接加入购物车
> **So that** 买到真正属于自己的商品，不需要离开这个网站，也不需要会设计软件

**典型操作路径**：
1. 浏览T恤商品页 → 点击「Add Your Design」
2. 编辑器弹窗 → 上传手机照片 → 自动去除背景
3. 拖拽调整位置 → 选择字体输入名字
4. 实时预览满意 → 「Add to Cart」
5. 结账，定制信息随订单自动传递给生产系统

### 支线故事3：仓库接收印刷任务

> **As a** ShipSage仓库操作员
> **I want to** 在WMS中看到带印刷文件的定制订单，系统自动生成印刷指令，按指令操作印刷设备完成质检和发运
> **So that** 定制订单的处理流程与普通订单一样标准化，不需要手动沟通

## 三、系统边界与角色

### 系统由三个子产品组成

```
子产品A：Design Studio（设计编辑器）
  - 商家端：创建商品模板、设置定制范围
  - 消费者端：在店铺内实时定制个性化商品
  - 技术形态：Web组件，嵌入Shopify商品页弹窗
          ↓ 订单+印刷文件

子产品B：Shopify App（商家管理后台）
  - 商品管理、定价、模板库管理
  - 订单管理、生产状态追踪
  - 数据报表、账单结算
          ↓ 生产指令

子产品C：生产履约（ShipSage OMS+WMS扩展）
  - 印刷文件管理、印刷队列调度
  - 自动选仓、复用现有SD出库流程
  - 质检、发运、物流追踪回传
```

### 角色权限矩阵

| 角色 | Design Studio | Shopify App | 生产履约 |
|------|--------------|-------------|----------|
| 消费者 | 在商家设定范围内定制 | 无权限 | 无权限 |
| 商家 | 配置模板和定制范围 | 全功能 | 只读查看状态 |
| 仓库操作员 | 无权限 | 无权限 | 执行印刷/发运 |
| 仓盛Admin | 全功能 | 全功能 | 全功能 |

## 四、验收标准 & NFR

### 功能验收标准

**Design Studio（消费者端）**
- [ ] AC1：消费者可上传PNG/JPG（≤20MB），5秒内完成上传和预览渲染
- [ ] AC2：支持文字输入，字体库≥20种，支持颜色、旋转、缩放
- [ ] AC3：印刷区域边界可视，超出区域自动高亮警告
- [ ] AC4：实时Mockup预览，操作响应延迟≤500ms
- [ ] AC5：点击「Add to Cart」后，设计图+位置+尺寸参数完整附到Shopify订单
- [ ] AC6：支持移动端H5（iPhone/Android主流机型），触屏操作流畅

**Shopify App（商家端）**
- [ ] AC7：商家安装后5分钟内可发布第一个定制商品
- [ ] AC8：Shopify新订单≤30秒推送到生产系统
- [ ] AC9：商家可实时查看每笔订单生产状态（待生产/生产中/已发运）
- [ ] AC10：利润计算器实时展示：售价-成本-运费=商家净利润

**生产履约（ShipSage对接）**
- [ ] AC11：定制订单自动生成符合印刷规格的高清文件（≥300DPI）
- [ ] AC12：文件低于150DPI自动警告并暂停订单，通知商家
- [ ] AC13：印刷任务自动分配到收货地最近仓库的印刷队列
- [ ] AC14：发运后物流追踪号自动回传Shopify，触发消费者通知邮件

### NFR（非功能性需求）

| 指标 | 要求 | 说明 |
|------|------|------|
| 设计器加载时间 | ≤3秒（4G网络）| 超过3秒消费者流失率>50% |
| 并发用户 | 支持1000并发设计操作 | 商家促销期峰值估算 |
| 印刷文件生成 | ≤10秒/单 | 下单后自动生成，不阻塞结账 |
| 数据安全 | 消费者上传图片加密存储 | GDPR/CCPA合规 |
| 移动端兼容 | iOS 14+、Android 10+ | 60%+消费者从手机下单 |
| 可用性SLA | 99.9%（月均）| 宕机直接影响商家销售 |

## 六、关键风险与约束

### 业务规则约束（来自ShipSage系统）

- 多货主隔离：每个商家的设计模板、订单数据、印刷文件必须严格隔离（company_id隔离）
- 出库SD流程：定制订单必须走完整SD流程，不可跳步
- 费用可追溯：每笔订单的印刷费+出库费+运费必须生成独立费用明细
- 金额精度：所有费用字段使用Decimal，禁止Float/Double
- 库存管理：空白品通过ASN入库，qty_available管理可用库存，订单分配时qty_allocated锁定

### 技术风险

| 风险 | 等级 | 应对措施 |
|------|------|----------|
| 设计器前端复杂度高，开发可能超期 | 高 | 采用Fabric.js成熟库，避免从零开发 |
| 消费者上传图片版权侵权 | 高 | AI审核+用户协议免责+DMCA处理流程 |
| 印刷文件质量不稳定 | 中 | 严格DPI检测+印前预检，低质量自动拦截 |
| Shopify App审核周期4-8周 | 中 | 提前准备合规文档，MVP先走私邀Beta |
| 仓内印刷设备采购周期长 | 中 | 提前与供应商谈判，或先外包印刷验证需求 |

## 八、专家建议

**1. 极简优先，不要做「功能完整但体验平庸」的设计器**

Printful/Printify的设计器功能完整但UX平平，消费者需要学习成本。真正的差异化在于「消费者3步完成定制」的极简体验：上传图片 → 调整位置 → 加入购物车。MVP只做这3步，做到极致，其他功能都是二期。

**2. 以「消费者定制」为核心，而非「商家设计工具」**

Riin Factory的失误是把精力放在商家设计工具上，最终变成了又一个Printful替代品，陷入同质化竞争。我们的核心护城河是「消费者在商家店铺内直接定制」，这才是真正的B2B2C模式，也是竞品无法快速复制的壁垒。

**3. 空白品备货是业务关键，必须与产品开发同步规划**

设计器做得再好，空白品断货就无法履约。建议MVP阶段在SG-ONT1立即备货：白色T恤（Bella+Canvas 3001）XS-2XL各50件、黑色各50件，共约500件。以$4/件采购成本计，备货成本约$2000，是可接受的验证投入。

**4. 第一批种子商家要亲自服务，不要等App Store自然流量**

MVP上线后，主动联系10-20个月销$5k-50k的Shopify卖家做白名单测试。这批种子商家的成功案例（转化率、交期、客户评价）将是后续在App Store、社交媒体增长的最核心素材。

**5. 与ShipSage的集成采用「渐进式」策略，避免过早工程化**

MVP阶段可以「半自动」运营：设计器生成订单 → 运营人工下载印刷文件 → 手动在ShipSage创建VAS工单。等流程完全跑通、日均订单量超过20单后，再投入工程资源做全自动化API对接。过早追求全自动化会让MVP延期，验证速度是第一优先级。
