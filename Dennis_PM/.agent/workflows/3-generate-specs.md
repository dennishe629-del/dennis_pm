# Workflow: Phase 3 - 规格生成 (Make Specs)

此工作流是**最后一步**。前两步的思考和设计已经完成，现在是"组装"和"输出"。

## 步骤 0: 输出PRD大纲并等待确认 ⚠️ 必须执行，不可跳过

> **Goal**: 在生成完整PRD正文之前，先输出目录大纲让用户确认范围，避免大量返工。

**执行步骤**：

1. 读取 Phase 2 的《核心方案设计草稿》和 `templates/prd-template-cn.md`。
2. 基于当前需求，输出本次PRD的大纲，格式如下：

```
## PRD 大纲确认

**需求名称**：XXX
**目标用户**：XXX
**核心目标**：一句话

### 本次PRD将包含以下章节：
1. 背景与目标
2. 用户故事
3. 功能说明（含业务规则 R01~RXX）
4. 流程图（Mermaid）
5. 页面线框图（ASCII）
6. 异常处理
7. 权限说明
8. 验收标准

### 本次暂不包含（超出范围）：
- XXX
- XXX

### 需要你确认的关键决策点：
1. XXX（选项A / 选项B）
2. XXX

> ✅ 请确认大纲无误后，我将开始生成完整PRD正文。
> 如有调整，请直接告知修改点。
```

3. **等待用户明确回复确认**（"可以"、"开始写"、"没问题"均视为确认）。
4. 用户确认后，进入步骤1生成完整PRD。
5. 用户如有修改意见，调整大纲后再次确认，**不得在未获确认前直接生成正文**。

---

## 步骤 1: 生成PRD (PRD Generation)

1. 读取 Phase 2 的《核心方案设计草稿》。
2. 读取 `templates/prd-template-cn.md`，严格按模板结构填充。
3. 确保所有业务规则都有编号 (R01, R02...)。
4. 确保包含"异常处理"和"权限说明"章节。

### ⚠️ 强制输出要求（不可跳过）

每个任务必须包含以下三个 section，缺一不可：

**1. 流程图（必须使用 Mermaid flowchart 语法）**
- 覆盖正向主流程
- 覆盖关键异常分支（拒绝/撤回/失败）
- 不允许用文字描述代替流程图

**2. 页面线框图（必须使用 ASCII 字符绘制）**
- 为每个涉及 UI 的功能点绘制关键页面布局
- 必须包含：页面标题区、操作按钮区、主要内容区（列表/表单/详情）
- 在线框图下方用「交互说明」描述关键交互行为
- 若一个任务有多个页面（列表页+详情页+表单弹窗），每个页面单独绘制

**3. 功能说明表格**
- 每行包含：功能点 | 描述 | 业务规则编号

### 线框图绘制规范


## 步骤 2: 生成前端Demo (Optional)

> **Goal**: 将PRD转化为可视化的代码。

如果需要前端Demo：

1. 阅读 `.agent/skills/frontend-designer/SKILL.md`，按其规范执行。
2. 根据PRD涉及的系统（Merchant Admin / ShipSage Admin / WMS / BI / Design Studio）选择对应UI风格。
3. 优先生成纯HTML + Tailwind单文件，可直接在浏览器打开，无需构建工具。
4. 多系统Demo必须生成 `index.html` 导航Hub页，各系统页面独立HTML文件。
5. 确保包含Mock数据、状态枚举映射、空状态/错误状态UI。

## 步骤 3: 交付与归档 (Delivery)

1. 将最终PRD保存到 `prds/[模块名]/PRD.md`。
2. 将Demo代码保存到 `prds/[模块名]/demo/`。
3. 输出一份《验收标准清单(AC List)》供测试使用。

**Done**: 任务完成！
